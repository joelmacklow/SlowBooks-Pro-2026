# PRD — Company logo copy in the top-right corner of financial reports

## Date
2026-04-30

## Objective
Add a top-right copy of the company logo to all generated financial report PDFs so they carry the company brand consistently across printed documents.

## Scope
### In scope
- invoice PDF
- estimate/quote PDF
- credit note PDF
- purchase order PDF
- customer statement PDF
- payroll payslip PDF
- general report PDF output generated by `app/services/pdf_service.py`
- shared PDF theme/layout helpers used by those templates

### Out of scope
- non-PDF application screens
- email HTML templates
- changing the existing left-side invoice logo treatment beyond the new right-side copy
- other document layout changes unrelated to the logo placement

## Requirements
1. Every financial PDF document should show the company logo in the top-right corner when a logo is configured.
2. The logo should be a copy, not a replacement for existing branding already present in some templates.
3. Existing layouts, tables, and content must remain stable apart from the header branding addition.
4. Templates without a configured logo should continue to render cleanly.
5. The implementation should be shared where possible to avoid duplicated markup drift.

## Implementation sketch
- Add a shared Jinja partial that renders the top-right logo block when `company.company_logo_src` exists.
- Add shared CSS for logo sizing and header layout alignment.
- Include the logo partial in every PDF document header.
- Update report-specific markup separately because it does not reuse the shared body template.
- Extend PDF formatting tests to assert the logo copy is rendered across representative document types.

## Risks
- Adding another branded element to the invoice header could crowd the layout if the logo is too large.
- The report template is structurally separate, so it must be updated in parallel with the shared PDF theme.
- The top-right logo should not overlap totals or document titles on small page widths.

## Acceptance criteria
- Financial PDFs render a company logo in the top-right corner when configured.
- Existing PDF formatting assertions continue to pass.
- The new logo block is shared or consistently applied across all PDF document families.
